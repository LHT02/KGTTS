KIGTTS 隐私政策

本页面适用于 Android 应用 KIGTTS（包名：com.lhtstudio.kigtts.app）。KIGTTS 默认以设备本地处理为主，除非用户主动下载、分享、打开外链或跳转第三方应用，否则不会把用户内容上传到开发者自建服务器。

1. 个人信息处理者

  • 应用名称：KIGTTS
  • 应用包名：com.lhtstudio.kigtts.app
  • 个人信息处理者 / 应用提供者：KIGTTS 项目维护者（LHTStudio）
  • 反馈渠道：可通过 KIGTTS 项目的公开仓库、发布页或应用发布渠道提交问题反馈。

2. 我们处理哪些数据

2.1 麦克风音频

当你启用语音识别、按住说话、开关模式、说话人验证、音频测试等功能时，应用会读取麦克风音频，用于本地实时语音识别、字幕上屏、TTS 前处理、语音增强、VAD、降噪、回声处理、说话人验证和音频测试。

这些音频默认在设备本地处理。KIGTTS 不会默认把麦克风音频上传到开发者自建服务器。

2.2 相机画面、图片和二维码内容

当你使用二维码扫描功能时，应用会访问相机画面；当你主动选择图片进行识别时，应用会读取你选择的图片，用于在本地识别二维码或条码内容，并根据二维码类型提示打开网页、复制内容或跳转第三方应用处理。

2.3 你主动导入、选择或创建的文件

当你使用内建文件管理器、系统文件选择器、系统图库、语音包安装、音效导入、预设导入导出、名片背景图设置等功能时，应用可能读取你主动选择的文件或目录。这类文件可能包括语音包、资源包、便捷字幕预设、音效板预设、音频文件、名片背景图、日志文件和导出的预设文件。

2.4 你在应用内创建或保存的内容

KIGTTS 会在设备本地保存你主动创建、编辑或触发的内容，例如便捷字幕分组、快捷文本、快捷名片、音效板配置、语音识别结果、字幕历史、说话人验证样本、悬浮窗设置、音频设置、资源包状态和应用日志。

2.5 已安装应用列表和快捷方式信息

当你添加悬浮窗启动器应用、配置第三方快捷方式或配置音量热键目标时，KIGTTS 可能读取设备中可启动应用的名称、包名、图标和系统公开的启动入口信息。这些信息默认仅在本地使用，不会上传到开发者自建服务器。

2.6 无障碍相关数据

当且仅当你主动开启“KIGTTS 音量热键辅助”无障碍服务后，应用会使用无障碍能力更稳定地监听音量键序列，并在你主动触发“QQ 扫一扫（无障碍）”时辅助识别 QQ 当前界面节点并执行必要点击或手势。KIGTTS 不会使用无障碍功能读取聊天内容、密码、支付密码或其它非本应用功能所需的信息。

2.7 蓝牙媒体标题字幕

如果你主动开启“蓝牙媒体标题字幕”功能，KIGTTS 会把当前大字幕内容写入 Android 系统媒体会话的标题信息，让部分蓝牙歌词屏、车机、小屏设备或系统媒体界面把当前字幕显示为媒体标题。该功能可能会覆盖其它媒体应用显示的标题信息。

2.8 权限状态、设备状态和运行日志

为了完成上述功能，KIGTTS 可能读取或记录权限状态、音频设备类型、音量变化状态、当前 TTS/ASR/VAD/语音包配置、应用运行日志和错误日志。日志默认保存在本地，只有当你主动分享日志时，日志内容才会发送给你选择的目标应用或联系人。

2.9 静默状态或后台运行时的信息处理

当你主动开启或正在使用相关功能时，即使 KIGTTS 处于后台、息屏、锁屏或悬浮窗折叠状态，应用仍可能在必要范围内继续处理相关信息。

  • 语音识别、按住说话、开关模式、说话人验证或后台保活：在功能运行期间继续读取麦克风音频、音频设备状态和识别配置，用于本地语音识别、字幕上屏、TTS 前处理、降噪、VAD、说话人验证和维持实时语音链路。
  • 悬浮窗、迷你控件和锁屏显示：在悬浮窗启用期间保存和更新悬浮窗位置、展开或折叠状态、当前显示文本、快捷字幕、快捷名片和音效板状态，用于在其它应用上方或锁屏场景继续显示和响应你主动开启的悬浮控件。
  • 音量热键：在你开启音量热键后，非无障碍模式会读取系统音量变化状态；无障碍稳定监听模式会在你授权后读取音量键事件，用于判断你配置的音量键序列并触发对应功能。
  • 实时通知和蓝牙媒体标题字幕：在你开启后，应用会把当前上屏大字幕、短状态和播放控制状态写入系统通知或系统媒体会话，用于通知栏、Live Updates、蓝牙歌词屏、车机或小屏设备显示。
  • 日志和故障排查：在功能运行或异常发生时，应用可能在本地记录运行状态、错误信息、权限状态和设备状态，用于排查崩溃、卡顿、导入失败或音频链路异常。

上述后台处理只在你主动开启相关功能或相关功能仍处于运行状态时发生，处理范围限于实现对应功能所必需的信息。KIGTTS 不会在后台默认把上述信息上传到开发者自建服务器，也不会将其用于广告画像、商业营销画像或与本应用功能无关的用途。你可以通过关闭对应功能、撤销系统权限、停止前台服务、清除应用数据或卸载应用来停止相关处理。

3. 我们如何使用这些数据

我们仅将上述数据用于实现你可见、可预期并主动启用的功能，包括本地语音识别、字幕显示、TTS 朗读、音频增强、说话人验证、二维码扫描、悬浮窗、热键、第三方快捷方式联动、蓝牙媒体标题字幕、本地日志记录和故障排查。

KIGTTS 不会将上述数据用于广告画像、商业营销画像或与本应用功能无关的自动化决策。

4. 数据是否会上传、共享或转让

默认情况下，KIGTTS 不提供开发者自建账号系统、云端同步系统、广告系统或统计后台，不会默认把你的音频、图片、文本、名片、音效、预设、说话人样本或应用列表上传到开发者自建服务器。

当你主动下载语音识别资源包、Kokoro 资源包或打开下载源时，应用会访问你选择的下载源，例如魔搭、Hugging Face、HF-Mirror 或你自定义的资源地址。下载源服务器可能接收必要的网络请求信息。

当你主动使用系统分享面板、打开二维码网址、跳转第三方应用、导出预设或分享日志时，相关数据会被发送给你选择的目标应用、网站或联系人。

KIGTTS 使用 Android 系统组件和若干第三方开源 / 本地功能组件来完成基础功能，例如 AndroidX、CameraX、ML Kit Barcode Scanning、ZXing、ONNX Runtime、sherpa-onnx、CanHub Image Cropper、Apache Commons Compress、XZ、Material Components 等。这些组件主要用于界面、相机预览、二维码识别、模型推理、图片裁剪、压缩包处理和本地音频能力。

KIGTTS 不接入第三方广告 SDK、统计 SDK、商业推送 SDK 或个性化推荐 SDK。应用不会通过第三方 SDK 在静默状态或后台运行时收集 IMEI、IMSI、设备 MAC 地址、SUPI、SUCI、精确位置、联系人、通话记录、日历、短信、本机电话号码等信息。

  • ONNX Runtime、sherpa-onnx：仅在语音识别、TTS、语音增强、VAD 等功能运行时参与本地模型推理，不用于上传音频或收集设备标识。
  • CameraX、ML Kit Barcode Scanning、ZXing：仅在你打开二维码扫描或主动选择图片识别时处理相机画面或图片内容，不在后台持续扫描。
  • CanHub Image Cropper：仅在你主动裁剪头像、名片图片等图片时处理所选图片。
  • Apache Commons Compress、XZ：仅在你主动导入、下载、安装或解压资源包、语音包、预设包时处理对应文件。
  • AndroidX、Material Components、MediaSession、通知和浏览器相关系统组件：仅用于界面显示、系统通知、蓝牙媒体标题字幕、Chrome Custom Tabs 或 WebView 等你主动启用的功能。

上述组件处理范围限于实现对应功能所必需的数据。KIGTTS 不会主动把这些数据提供给第三方广告、统计、营销或画像服务。

KIGTTS 不会出售你的个人信息，也不会在未经你同意的情况下公开披露你的个人信息。若因法律法规、行政或司法机关依法要求必须提供相关信息，我们会在符合法律要求的范围内处理。

KIGTTS 不提供广告、个性化推荐或定向推送功能，也不会使用第三方来源的个人信息进行精准营销。

6. 你的选择与控制

  • 在系统设置中关闭麦克风、相机、媒体读取、通知、悬浮窗或无障碍权限。
  • 在应用设置中关闭蓝牙媒体标题字幕、音量热键、无障碍稳定监听、快捷文本触发音效等可选功能。
  • 在应用内删除便捷字幕、名片、音效、语音包、资源包、历史记录、日志或预设。
  • 使用系统设置清除 KIGTTS 应用数据，或直接卸载 KIGTTS。
  • 删除你在外部目录中保存或导出的文件。
由于 KIGTTS 默认不建立开发者服务器端账户和云端存储，我们通常无法代你远程删除设备本地数据。你可以直接在设备本地完成删除。

7. 数据存储和保留

KIGTTS 会把设置、预设、音效、名片、日志、导入资源和部分运行状态保存在设备本地应用目录或你主动选择的目录中。这些数据通常会保留到你在应用内手动删除、清除应用数据、卸载应用或删除外部文件为止。

如果你将文件导出到外部目录或分享给第三方应用，导出后的副本将不再受 KIGTTS 应用内删除操作控制。

8. 数据安全

KIGTTS 尽量按最小必要原则设计功能：核心语音识别、字幕、TTS、音频处理和二维码识别以本地处理为主；权限按功能需要请求；无障碍功能默认关闭；不默认接入开发者自建云端收集系统、广告系统或统计后台；导入、导出、分享等跨应用行为需要你主动触发。

但任何移动设备、操作系统、第三方应用、蓝牙设备、存储介质或网络环境都不可能绝对零风险。请根据自己的设备安全状态谨慎使用相关功能。

9. 未成年人说明

KIGTTS 不以未成年人为主要目标用户。如未成年人需要使用本应用，建议在监护人知情和指导下使用。监护人可以通过系统权限管理、应用内删除、清除应用数据或卸载应用来管理相关内容和权限。

10. 隐私政策的更新

如果 KIGTTS 的功能、权限使用方式、数据处理方式或第三方组件发生实质变化，我们可能更新本隐私政策，并在应用内、仓库文档、网页版或发布说明中同步更新。

更新后的版本以最新发布日期为准。继续使用相关功能，即表示你已了解更新后的政策内容；如果你不同意相关变化，可以停止使用对应功能、关闭权限或卸载应用。
